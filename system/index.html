06 / Philosophy

How the system is run

These are the operating rules that matter for public client work.

  • 01
    Human authority is final. No AI decides what gets built, shipped, or delivered. Every significant action requires human approval. Agents propose; humans decide.
  • 02
    Context is written down. Project constraints, decisions, and confirmed approaches are stored so the next session has a real starting point.
  • 03
    Credentials never touch the repo. API keys, tokens, and secrets stay in local environment files outside the repository. A pre-commit hook scans every commit. No exceptions, no workarounds.
  • 04
    Simplicity beats flexibility. Write the minimum code that solves the problem. No abstractions for single-use code. No frameworks unless the complexity earns them. If it's 200 lines and 50 would work — rewrite it.
  • 05
    Truth over polish. No fake testimonials, fake client lists, fake numbers, or inflated claims. If something is a placeholder or work in progress, it is labeled that way.
