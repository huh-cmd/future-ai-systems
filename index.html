Process

How work moves from idea to shipped file

Every build follows the same path: clear intake, wireframe, Claude Code build, mobile pass, and human review before anything is delivered.

01

Intake

You fill a brief: goal, audience, tone, references. Takes 10 minutes.

02

Wireframe

Sections and hierarchy defined in text. You approve before build starts.

03

Build

Claude Code generates clean HTML/CSS/JS against the wireframe. No frameworks.

04

Mobile pass

375px baseline, tested at 768px and 1280px. Ships responsive.

05

Deliver

Live GitHub Pages URL + clean source files. One free revision and clear handoff included.

Delivery

What you receive

The handoff is designed to be simple: you receive the finished page as clean files in a zip package. Preview links are optional during review, but the zip is the durable delivery.

ZIP

Source package

Final HTML/CSS/JS files packaged with any images or assets needed for the page.

DOC

Customer README

A short handoff note explains what each file does and how to make basic text or image edits.

QA

Mobile-ready page

Reviewed for common phone and desktop widths before delivery.

SAFE

No keys required

Normal static landing pages do not need API keys, passwords, databases, or customer login access.
