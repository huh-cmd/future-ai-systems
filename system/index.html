05 / Philosophy

How the system is run

Alexandria isn't just tooling — it has operating principles that govern how AI is used, when to escalate, and what stays private. These are the ones that directly affect client work.

  • 01
    Human authority is final. No AI decides what gets built, shipped, or delivered. Every significant action requires human approval. Agents propose; humans decide.
  • 02
    Nothing gets lost between sessions. Alexandria carries context forward. Every project, constraint, decision, and confirmed approach is stored and reloaded. You never re-explain context.
  • 03
    Credentials never touch the repo. API keys, tokens, and secrets live in one place: .env.keys. A pre-commit hook scans every commit. No exceptions, no workarounds.
  • 04
    Simplicity beats flexibility. Write the minimum code that solves the problem. No abstractions for single-use code. No frameworks unless the complexity earns them. If it's 200 lines and 50 would work — rewrite it.
  • 05
    Data over stories. Token usage is tracked. Build times are logged. Turnaround is measured. The numbers in this system are real — not marketing estimates.
